# Security Policy

## Supported Scope

Security reports are welcome for the template source, documentation, and shipped integration examples.

## Please Do Not

- open a public issue for a sensitive vulnerability
- include real credentials, API keys, or user data in reports

## Reporting

If you discover a security issue, contact the maintainers privately and include:

- a clear description of the issue
- affected files or behavior
- reproduction steps
- potential impact

## Notes For Template Users

- This template ships with a disabled contact form by default.
- Do not commit production secrets.
- Review any custom API endpoint, serverless function, or third-party integration you add on top of the template.
